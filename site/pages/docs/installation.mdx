# 安装

通过你的包管理器、`<script>` 标签或从源代码构建来安装 Viem。

## 包管理器

安装所需的包。

:::code-group
```bash [pnpm]
pnpm add viem
```

```bash [npm]
npm install viem
```

```bash [yarn]
yarn add viem
```

```bash [bun]
bun add viem
```
:::

## CDN

如果你不使用包管理器，你也可以通过 ESM 兼容的 CDN 使用 Viem，例如 [esm.sh](https://esm.sh)。只需在 HTML 文件底部添加一个 `<script type="module">` 标签，内容如下。

```html
<script type="module">
  import { createClient } from 'https://esm.sh/viem'
</script>
```

## 使用未发布的提交

如果你迫不及待想测试最新功能，可以从 `canary` 标签安装（跟踪 [`main`](https://github.com/wevm/viem/tree/main) 分支）。

:::code-group
```bash [pnpm]
pnpm add viem@canary
```

```bash [npm]
npm install viem@canary
```

```bash [yarn]
yarn add viem@canary
```

```bash [bun]
bun add viem@canary
```
:::

或者将 [Viem 仓库](https://github.com/wevm/viem)克隆到本地机器，构建并自行链接。

```bash
gh repo clone wevm/viem
cd viem
pnpm install
pnpm build
pnpm link --global
```

然后转到你使用 Viem 的项目，运行 `pnpm link --global viem`（或你用于全局链接 Viem 的包管理器）。

## 安全

与以太坊相关的项目常常成为攻击的目标，以窃取用户资产。确保你遵循项目的安全最佳实践。以下是一些快速入门的建议。

- 固定包版本，谨慎升级，并检查锁定文件的更改，以最小化 [供应链攻击](https://nodejs.org/en/guides/security/#supply-chain-attacks) 的风险。
- 安装 [Socket Security](https://socket.dev) [GitHub 应用](https://github.com/apps/socket-security)，以帮助检测和阻止供应链攻击。
- 添加 [内容安全策略](https://cheatsheetseries.owasp.org/cheatsheets/Content_Security_Policy_Cheat_Sheet.html)，以防止外部脚本在你的应用中运行。
